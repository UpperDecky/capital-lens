# Capital Lens -- Development Guidelines

## Project Overview

Capital Lens is a self-hosted, real-time financial intelligence platform that fuses corporate filings,
congressional trades, geopolitical conflicts, aircraft/maritime tracking, satellite fire detection,
prediction markets, and Telegram OSINT into one AI-enriched feed. Every event gets plain-English
summaries, market impact assessments, and investment signals generated by Claude via OpenRouter.

**Architecture**: FastAPI backend (SQLite) + React 18/Vite frontend + APScheduler background jobs.

**Stack**: Python 3.11, FastAPI 0.111, SQLite3, APScheduler 3.10.4, React 18, Vite, Tailwind CSS,
D3.js, OpenRouter API (for enrichment), Pyrogram (Telegram OSINT).

## CRITICAL RULES -- READ FIRST

### Rule 1: No Em-Dashes or Multi-Byte Unicode in Python Files

**NEVER** use em-dashes (--), arrows (->), curly quotes, or any non-ASCII character in .py files.
Windows NTFS filesystem + Linux bash causes silent file truncation at those characters.

WRONG:
```python
# Fetch events -- returns list
```

CORRECT:
```python
# Fetch events -- returns list
```

Replace with: `--` (double hyphen) or ` - ` (space-hyphen-space).

### Rule 2: Use Bash Heredoc for All Multi-Line File Writes

When writing or rewriting .py files from bash, ALWAYS use heredoc with PYEOF delimiter:

```bash
cat << 'PYEOF' > backend/some_file.py
"""Module docstring."""
import os

def my_function() -> None:
    pass
PYEOF
```

Never use Edit tool for files >50 lines -- use heredoc rewrites.

### Rule 3: No SQLite WAL Mode

The database lives on a Windows NTFS mount. WAL mode creates -shm/-wal sidecar files
that cause corruption and sync failures in this environment.

WRONG:
```python
conn.execute("PRAGMA journal_mode=WAL")
```

CORRECT: Do nothing (SQLite defaults to DELETE/rollback journal, which is fine).

### Rule 4: Enrichment Only From Scheduler

The Claude/OpenRouter enrichment pipeline is slow (4s/event) and expensive. It must NEVER
be called from HTTP routes. It runs only inside APScheduler jobs.

WRONG:
```python
@app.get("/feed")
async def get_feed():
    enrich_pending_events(conn)  # NEVER DO THIS
    ...
```

CORRECT: Routes query already-enriched events. Enrichment runs in scheduler every 5 minutes.

## Ingestor Pattern

All ingestors follow this structure:

```python
"""
Ingestor for [Source Name].
Pulls [data description] every [interval].
API key: [KEY_NAME] (optional/required)
"""
import uuid
from datetime import datetime, timezone
import httpx
from backend.database import get_connection
from backend.config import API_KEY_VAR

def fetch_[source]_data(conn, entity_map: dict) -> int:
    """Fetch and store data. Returns count of new records inserted."""
    if not API_KEY_VAR:
        return 0  # Skip gracefully if no key

    inserted = 0
    try:
        resp = httpx.get("https://api.example.com/data", timeout=15)
        resp.raise_for_status()
        data = resp.json()
    except Exception as exc:
        print(f"[Ingestor] Error fetching: {exc}")
        return 0

    for item in data:
        try:
            event_id = str(uuid.uuid4())
            conn.execute(
                "INSERT OR IGNORE INTO events (id, entity_id, event_type, ...) VALUES (?,?,?,...)",
                (event_id, entity_id, "news", ...)
            )
            inserted += conn.execute("SELECT changes()").fetchone()[0]
        except Exception as exc:
            print(f"[Ingestor] Error storing item: {exc}")

    conn.commit()
    return inserted
```

Key patterns:
- Return 0 and skip gracefully if API key missing
- Use INSERT OR IGNORE for deduplication
- Wrap each item in try/except (don't let one bad record stop the batch)
- Return count of inserted records (scheduler prints this)
- All timestamps as UTC ISO strings: `datetime.now(timezone.utc).isoformat()`

## Enrichment Pipeline

Located in `backend/enrichment.py`. Uses OpenRouter API (OpenAI SDK pointing at openrouter.ai).

Models (priority order):
1. nousresearch/hermes-3-llama-3.1-405b:free
2. meta-llama/llama-3.3-70b-instruct:free
3. openrouter/free (auto-router)

Generates per event:
- plain_english: 1 sentence, 12-year-old level
- market_impact: 2 sentences, industry effects
- invest_signal: 2 sentences, investor implications + adjacent plays
- for_you: 2 sentences, personal takeaway
- sector_tags: JSON array of 3-5 tags
- importance: integer 1-5 (5 = market-moving)
- analysis: extended JSON with relationships, affected_sectors, timeline, context

Batch size: 5 events per run. Runs every 5 minutes via scheduler.
Throughput: ~18 events/hour on free tier.

## Common Gotchas

1. **Em-dash truncation**: Files written from bash with em-dashes get silently truncated at that
   character on Windows NTFS mount. Always use ASCII-only in Python files.

2. **Stale .pyc cache**: After rewriting a file, Python may use cached bytecode. Run with
   `python3 -B` to disable bytecode caching during testing.

3. **SQLite "unable to open database"**: The live DB is on Windows mount; the Linux sandbox
   cannot write to it directly. Test schema changes against `:memory:`.

4. **maritime.py uses asyncio**: The AIS WebSocket ingestor runs
   `asyncio.run(_collect_messages_async(...))` inside a sync APScheduler job. This is intentional.

5. **Telegram requires one-time setup**: Run `python -m backend.ingestors.telegram_osint --setup`
   once to generate the TELEGRAM_SESSION string. Add it to .env.

6. **Frontend npm peer deps**: Always use `npm install --legacy-peer-deps` due to D3 v7 compat.

7. **No WAL**: If you see journal_mode=WAL anywhere in new code, remove it.
